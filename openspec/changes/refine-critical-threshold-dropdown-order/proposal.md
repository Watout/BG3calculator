# Proposal

## Why

攻击项界面里的紧凑数值下拉已经统一承载模板执行次数、主手执行次数、副手执行次数和重击阈值，但还有两个可用性问题：

- 重击阈值默认从 `1+` 往上排，和玩家通常从高阈值向下浏览的心智不一致。
- 共享菜单会显示右侧滚动条，视觉上偏拥挤，也会让同一类紧凑控件看起来不够统一。

## What Changes

- 让重击阈值下拉在固定范围 `1..20` 内按 `20+ -> 1+` 的降序展示。
- 保持模板执行次数、主手执行次数、副手执行次数沿用现有升序滑动窗口。
- 隐藏紧凑数值下拉菜单的可视滚动条，但保留滚轮与滚动驱动的窗口滑动行为。
- 补充窗口算法、界面交互和文档说明。

## Impact

- 影响范围限于 `apps/desktop-tauri` 的紧凑数值下拉实现与相关测试。
- 不改变 `compute.worker.ts`、领域计算或 `1+` 的天然 `1` 自动失手语义。
