# Proposal

## Why

桌面端攻击项表单里已经有一部分下拉输入切到了紧凑按钮式弹层风格，例如模板执行次数、主副手执行次数和重击阈值；但“攻击掷骰状态”“目标伤害修正”“伤害骰模式”仍是原生 `<select>`，视觉和交互不一致。

## What Changes

- 为有限枚举选项补一套与现有紧凑下拉一致的弹层式组件。
- 将攻击掷骰状态、目标伤害修正、伤害骰模式切换到统一的紧凑下拉触发器、菜单和选项样式。
- 保持这三个字段的业务语义不变，只统一展示与交互方式。
- 补充界面测试与文档说明。

## Impact

- 影响范围限于 `apps/desktop-tauri` 的表单 UI 组件和测试。
- 不改变 `compute.worker.ts`、领域计算或现有枚举值。
