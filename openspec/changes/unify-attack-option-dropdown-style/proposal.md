# Proposal

## Why

桌面端攻击项表单已经把模板执行次数、主手/副手执行次数和重击阈值切到了紧凑按钮式下拉，但“攻击掷骰状态”“目标伤害修正”“伤害骰模式”仍然保留原生 `<select>`，导致同一块表单里出现两套明显不同的下拉视觉与交互。

## What Changes

- 为枚举型选项补一个共享的紧凑下拉组件。
- 将攻击掷骰状态、目标伤害修正、伤害骰模式切换到和重击阈值一致的 trigger / menu / option 样式。
- 保持这些字段原有的业务值和计算语义不变，只替换展示与交互外壳。
- 更新相关测试、文档和 openspec 记录。

## Impact

- 影响范围在 `apps/desktop-tauri/src/App.tsx`、现有样式类复用、以及桌面端表单交互测试。
- 不影响 `compute.worker.ts`、领域计算逻辑或重击阈值的数值窗口算法。
